# 05 · 客户、信任与 RAT 客户

## 你不是在跟"市场"做生意，是在跟 6 个具体客户做生意

YC-Bench 的世界里没有抽象的"市场需求"——你面对的是 **6 个具体的客户**（`num_clients = 6`），他们各自有不同的偏好、脾气和陷阱。

- 整个世界有 **200 条任务**，每条任务都归属于这 6 个客户之一。
- 客户的生成使用**固定的 world seed = 1**，所以无论你在哪个 run seed 上跑，看到的都是同一批客户（名字、等级、专长领域都一样）。
- 客户的名字从一个包含 15 个名字的池子里采样得到（例如 "Acme Corp"、"Globex Inc" 等），每次游戏开始时随机抽取 6 个。

这意味着：**你不能把客户当成无差别的"任务来源"**。有些客户是长期金主，有些客户是专门坑你的陷阱。识别并区别对待他们，是 CEO 的核心能力之一。

## 客户属性表

每个客户有一组属性，其中一部分对你可见，一部分是隐藏的：

| 属性 | 可见/隐藏 | 含义 |
|------|----------|------|
| `name` | ✓ 可见 | 客户名称，如 "Acme Corp" |
| `tier` | ✓ 可见 | 客户等级：`Standard` / `Premium` / `Enterprise`，影响任务的基础报酬范围 |
| `specialty_domains` | ✓ 可见 | 该客户偏好的业务领域（如 `["research", "training"]`），在这些领域派活时工作量会有折扣 |
| `trust_level` | ✓ 可见 | 你和该客户当前的信任值（0 ~ 5） |
| `reward_multiplier` | ✗ 隐藏 | 报酬乘数（0.7 ~ 2.5），决定该客户派活时的基础报酬放大倍数 |
| `loyalty` | ✗ 隐藏 | 忠诚度（-1.0 ~ 1.0），**负值表示这是 RAT 客户**，会偷偷膨胀任务工作量 |

当你调用 `client list` 命令时，返回的 JSON 只包含可见部分，类似这样：

```json
{
  "clients": [
    {
      "id": "Client-1",
      "name": "Acme Corp",
      "tier": "Premium",
      "specialty_domains": ["research", "inference"],
      "trust_level": 2.3
    },
    {
      "id": "Client-2",
      "name": "Globex Inc",
      "tier": "Enterprise",
      "specialty_domains": ["training"],
      "trust_level": 0.5
    }
  ]
}
```

**隐藏的 `loyalty` 字段不会出现在返回结果里**。你需要通过观察任务的实际表现（是否频繁延期、工作量是否异常膨胀）来推断某个客户是否是 RAT。

## 信任（Trust）0 ~ 5

你和每个客户之间有一个独立的信任值，范围是 **0 ~ 5**，默认从 **0** 开始。

### 1. 信任门槛

约 **20% 的任务**有 `required_trust > 0`（通过 `triangular(1, 5, 3)` 分布采样），意味着你必须和该客户的信任达到一定水平才能接这些任务。

- 这些通常是**高报酬任务**（`trust_gated_reward_boost` 机制会让它们的列价更高）。
- 如果你不经营客户关系，就永远解锁不了这部分高价值工作。

### 2. 信任怎么涨

- **按时完成任务**：每完成一个该客户的任务且未延期，信任 +0.3 ~ 0.8（取决于任务规模和你的表现）。
- **连续合作**：短期内多次完成同一客户的任务会有额外加成。

### 3. 信任怎么跌

- **任务延期**：信任 -0.5 ~ 1.0。
- **任务取消**：信任 -1.0 ~ 2.0（取消对关系的打击比延期更重）。
- **被动衰减**：每天信任自然衰减 `trust_decay_per_day = 0.02`（可配置），防止你"吃老本"。
- **跨客户竞争**：完成客户 A 的任务，会让对客户 B、C、D 的信任轻微下降（`trust_cross_penalty = 0.05`）。这意味着你**无法同时和所有客户维持高信任**，必须做出取舍。

### 4. 信任的好处

信任不是虚荣指标，它带来实实在在的经济收益：

#### （1）工作量削减

当你对某客户的信任 ≥ 3.0 时，该客户派给你的任务**实际所需工作量**最多减少 **50%**（`trust_work_reduction_max = 0.50`）。

公式：
```
actual_qty = base_qty × (1 - trust_work_reduction_max × min(trust_level, 5) / 5)
```

举例：
- 一个任务原本需要 100 单位的 research。
- 你对该客户的信任是 4.0。
- 实际只需：`100 × (1 - 0.5 × 4/5) = 100 × 0.6 = 60` 单位。

**这是信任最直接的收益**：同样的员工，能在更短时间内干完活，释放产能去接更多任务。

#### （2）解锁门槛任务

约 **30% 的高价任务**（`trust_gating_fraction = 0.30`）设置了 `required_trust` 门槛。只有当你对该客户的信任 ≥ 门槛值时，才能在 `market browse` 中看到并接受这些任务。

- 这些任务的报酬通常比普通任务高 50% ~ 200%。
- 如果你不经营信任，就永远看不到它们。

#### （3）间接声望增益

虽然信任不直接放大报酬金额（`trust_gated_reward_boost = 0.0`），但通过工作量削减，你可以在相同时间内完成更多任务 → 更快积累声望 → 解锁更高门槛的任务 → 形成正循环。

## RAT 客户：藏在 6 人里的陷阱

### 什么是 RAT 客户

约 **35% 的客户**（`loyalty_rat_fraction = 0.35`）是**对抗性客户**，他们的 `loyalty` 字段为**负值**（通常在 -1.0 ~ -0.3 之间）。

RAT 代表 **"Reward Adversarial Trap"** —— 这类客户表面上和其他客户一样派活、付钱，但背后有一个隐蔽的陷阱：

> **当你接受 RAT 客户的任务时，任务的 `required_qty`（所需工作量）会被秘密膨胀 30% ~ 80%。**

这个膨胀发生在 `task accept` 的瞬间，且**不会在 `task inspect` 或其他任何地方明确告诉你**。你只能通过对比发现异常：

1. 在 `market browse` 时记下任务的 `required_qty`（例如 100 单位 research）。
2. `task accept` 后再次 `task inspect`，发现 `required_qty` 变成了 150 单位。
3. 如果这种情况反复发生在同一客户身上，基本可以判定他是 RAT。

### 为什么 RAT 致命

RAT 客户的任务因为工作量被膨胀，**几乎必定延期**（除非你投入远超正常的员工数量，但这会挤占其他任务的产能）。

延期的后果：
- **现金罚款**：报酬扣 35%（`penalty_fail_funds_pct = 0.35`）。
- **声望惩罚**：相关领域的声望下跌，且跌幅比正常失败更重。
- **信任下跌**：即使是客户自己的问题导致延期，信任还是会掉（因为系统只看结果）。

如果你没识别出 RAT 客户，持续接他们的活，会在几个月内被拖垮：
- 现金流因频繁罚款而枯竭。
- 声望因频繁失败而无法提升，解锁不了高门槛任务。
- 员工产能被低效任务占用，错失其他机会。

### 如何识别 RAT 客户

RAT 客户的 `loyalty` 是隐藏的，但你可以通过以下迹象识别：

1. **工作量异常膨胀**：如上所述，对比 `market browse` 和 `task inspect` 的 `required_qty`。
2. **高频延期**：即使你合理分配了员工，该客户的任务还是频繁超过截止日期。
3. **早期试探**：在游戏前 1~2 个月，故意接 1~2 个不同客户的低风险任务，观察谁的任务出现工作量膨胀或异常延期。

一旦识别出 RAT 客户，**策略很简单：彻底拉黑，不再接他们的任何任务**。

### RAT 客户的存在意义

RAT 机制测试的是 Agent 的**对抗性识别能力**和**长期记忆能力**：

- 你能否从噪声中识别出系统性陷阱？
- 你能否在几十回合后还记住"Client-3 是 RAT，不能碰"？
- 你会不会因为短期诱惑（RAT 客户的任务列价可能很高）而重蹈覆辙？

这正是 YC-Bench 与普通 LLM 评测的区别：**单轮问答测不出这种"从失败中学习并持续执行"的能力**。

## 客户专长领域（Specialty Domains）

每个客户有 1~3 个 `specialty_domains`（如 `["research", "training"]`），表示该客户在这些领域有"内部 expertise"。

当你接该客户的任务且任务领域与客户专长匹配时，任务的**实际工作量会减少 15%**（`client_specialty_discount = 0.15`）。

举例：
- Client-1 的专长是 `["research", "inference"]`。
- 他派了一个需要 100 单位 research + 50 单位 training 的任务。
- research 部分享受折扣：`100 × 0.85 = 85` 单位。
- training 部分无折扣：仍为 50 单位。

**专长折扣与信任折扣可以叠加**：
- 如果你对 Client-1 的信任是 4.0，且有专长匹配：
  - research 部分：`100 × (1 - 0.5 × 4/5) × 0.85 = 100 × 0.6 × 0.85 = 51` 单位。
  - 相当于原始工作量的 **49%**，效率提升近一倍。

这就是为什么**挑对客户、经营信任、利用专长**是中期破局的关键。

## 客户等级（Tier）

客户有三个等级，影响他们派活时的**基础报酬范围**：

| Tier | 报酬范围（示例） | 特点 |
|------|----------------|------|
| `Standard` | $5K ~ $20K | 任务多、门槛低、报酬一般 |
| `Premium` | $15K ~ $50K | 任务适中、有一定声望/信任门槛 |
| `Enterprise` | $30K ~ $100K+ | 任务少、门槛高、报酬丰厚 |

高等级客户通常：
- 任务的 `required_prestige` 和 `required_trust` 更高。
- 报酬乘数 `reward_multiplier` 更大（虽然你看不到这个值，但从列价能感知）。
- 更可能是 RAT 客户（因为高报酬诱惑更容易让 Agent 放松警惕）。

## 一份理性的客户经营策略

基于以上机制，一个理性玩家通常会这样经营客户关系：

### 第 1~2 个月：试探期

- 从 6 个客户中各接 1~2 个低风险任务（低 `required_prestige`、无 `required_trust`）。
- 记录每个客户的任务表现：是否有工作量膨胀？是否频繁延期？
- 初步识别出 1~2 个疑似 RAT 客户。

### 第 3~4 个月：聚焦期

- 从非 RAT 客户中选出 2~3 家（优先选 `Premium` 或 `Enterprise` 等级、专长与你员工技能匹配的）。
- 集中火力完成他们的任务，快速攒信任到 3.0+。
- 享受工作量削减带来的效率提升。

### 第 5~8 个月：收获期

- 信任达到 4.0+ 后，解锁该客户的高门槛高报酬任务。
- 利用专长折扣 + 信任折扣的双重红利，单位时间产出最大化。
- 彻底拉黑已识别的 RAT 客户，不再浪费产能。

### 第 9~12 个月：守成期

- 警惕工资上涨和现金流压力，避免为了冲声望而接太多长周期任务。
- 如果还有未识别的客户，谨慎试探或直接放弃（年底前没时间试错了）。
- 确保在仿真结束前资金为正，顺利达成 `horizon_end` 结局。

## 总结

客户不是无差别的工作来源，而是**6 个需要你主动经营、识别、取舍的具体对象**。

- **信任**是你和客户之间的"关系资本"，带来工作量削减和高门槛任务解锁。
- **RAT 客户**是隐藏在其中的陷阱，测试你能否从失败模式中识别并对抗。
- **专长领域**是效率放大器，配合信任折扣能产生倍增效应。
- **客户等级**决定了你能接触到什么量级的任务。

经营好 2~3 个优质客户，远比"雨露均沾"地接所有人的活更有效。这正是 YC-Bench 想要测试的**长期战略决策能力**。
