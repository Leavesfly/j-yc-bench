# 08 · 核心策略张力与常见坑

j-yc-bench 里**没有最优解**——只有一组**互相拉扯的张力**，你怎么权衡决定了一年下来是赚钱还是破产。

## 张力 1：接活速度 vs 工资增长

每完成一个任务，**所有被指派员工**按 tier 中位数永久加薪：junior +$30/月，mid +$70/月，senior +$125/月。

→ **坑**："多接活＝多赚钱"是错觉。如果每单都"全员上"（8 人），8 人都涨薪，但 Brooks's Law 下第 5~8 人零贡献——**净效果是 4 人产出对应 8 人工资增长**。

→ **对策**：用最小有效团队（≤4 人）做每个任务。

## 张力 2：聚焦 vs 并行

- **聚焦**（1 员工 1 任务）：那个任务推进最快，但同时只前进 1 单。
- **并行**（1 员工 N 任务）：吞吐量切分 `base/N`，每个任务都慢 N 倍。

→ **坑 1**："总产能不变，多并行不亏"。**会亏**——并行让每个任务更晚完成，截止日期更容易翻车。

→ **坑 2**："专攻一单最稳"。**会浪费**——剩余员工白领工资，收入流速被限死。

→ **对策**：**2~4 个并行任务**是甜点。每个 1~3 核心员工，总切分不超过 2~3 倍。

## 张力 3：声望增长 vs 工作量爆炸

`reward_prestige_scale = 0.30`：高声望任务报酬 ×2.2（prestige=5）。
`prestige_qty_scale = 0.30`：工作量**也** ×2.2。

→ **坑**："prestige=5 报酬 2.2 倍，肯定值得"。**不一定**——时间也 2.2 倍，单位时间收入未必更高。

→ **对策**：以"单位时间收入"而非绝对挂牌价作为筛选指标。

## 张力 4：贪婪接活 vs RAT 识别

`market browse` 默认按报酬降序——**最贵的在最前面**。但 RAT 客户的 `reward_multiplier` 被强抬到 ≥1.875——**最贵的活反而最可能来自 RAT**。

→ **坑**：闭眼抢最贵的 → 连续撞 RAT → scope creep → 必定延期 → 三重打击。

→ **对策**：开局每个客户至少接一单建立观察样本，用 `client history` 看延期率，疑似 RAT 写进 scratchpad 黑名单。

## 张力 5：广撒网 vs 深耕单一客户

- **广撒网**：信任分散，吃不到工作量削减（`trust_work_reduction_max = 0.50`）。
- **深耕单家**：trust 涨到 5 工作量减半。但 `trust_cross_client_decay = 0.018`/单 + `trust_decay_per_day = 0.009` 双重衰减。

→ **对策**：**锁定 2~3 个好客户**轮流耕作，让每个涨到 3~5。

## 张力 6：现金流当下 vs 跑道未来

工资每月第一个工作日扣，`< 0` 立即破产。

→ **坑**：月末账上 $30,000、月薪 $24,000，觉得"稳"——但在做的任务要 20 天后才回款，下月初先扣工资。

→ **对策**：永远盯 `runway_months`，**不能让它在月底低于 1.0**。

## 张力 7：scratchpad 信息密度 vs token 成本

scratchpad **每回合注入系统 prompt**，每回合都吃 token。

→ **坑**：往里塞流水账 → 几十回合后 scratchpad 巨大 → API 调用又贵又慢。

→ **对策**：写"决策规则"不写"事件日志"。✅ "`Cipher Corp` = RAT，永不接"。❌ "Turn 15: 接了 Task-23"。

## 张力 8：探索 vs 利用

前期不知道谁是 RAT，必须探索。但每次探索有成本：接到 RAT → 失败 + 扣声望 + 扣 35% 报酬。

→ **对策**：开局**只探索低风险任务**（低价、低 prestige、小工作量），用最小代价获取信号。

## 张力 9：升声望 vs 差异化任务

声望涨上去解锁更高 prestige 任务（更贵但工作量也更大）。长期不动永远只能接便宜活。

→ **对策**：早期啃几个 prestige 3~4 的任务把声望抬到 2~3，之后大部分活都能解锁。

## 5 个最常见的"必败操作"

1. **每个任务都派 8 人**——payroll 增速是甜点策略的 ~2.7 倍。
2. **闭眼接 reward 最高的活**——直接撞 RAT 陷阱。
3. **跟 6 个客户都做一点**——信任全卡在 1 以下，零红利。
4. **连续接活不歇**——单位时间收入低于 payroll 增速时在亏钱。
5. **不写 scratchpad**——20 回合后忘了谁是 RAT，又上当。

## 反直觉结论

> 在 j-yc-bench 里，**"什么都不做"的策略意外地不算最差**。先用小团队（3 人）接 1~2 个低风险任务，宁可慢一点也比"贪婪全押"安全。因为：payroll 增长是永久的，探索 RAT 的代价也是永久的。**早期的每一个错误决定都会被复利放大**。
