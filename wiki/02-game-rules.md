# 02 · 游戏规则总览：CEO 这一年要干什么

这一篇用"玩家手册"的口吻，把整个游戏规则讲一遍。读完你应该能在脑子里完整地想象出一局是怎么走的。

## 起手牌

游戏开始时（以仓库中现有的 `default` 预设为例），你（Agent）拿到的资源是这样的：

- **一家公司**：默认名为 `BenchCo`，启动资金 $200,000。
- **8 名员工**：tier 比例由配置控制（`share` 字段：junior 0.50、mid 0.35、senior 0.15），实际由一个 10 人的固定 tier 模板（5 个 junior、3 个 mid、2 个 senior）截取前 8 个并随机洗牌得到。每人有工资。
- **一张任务市场**：200 个任务排队等你接。
- **一批客户**：默认 6 家。每个任务都有所属客户。其中约 35% 是**藏着对抗性意图**的 RAT 客户（`loyalty_rat_fraction = 0.35`）。
- **一段固定时间**：仿真从 `2025-01-01` 开始，时长由 `horizon_years` 决定（默认 1 年）。

你**看得到的东西**（CLI 直接返给你）：

- 每个员工在每个领域的真实速率（`employee list` 返回 `skill_rates`）。
- 每个客户的"等级"与你和他的当前信任值。

你**看不到的东西**（设计上对 Agent 隐藏）：

- 客户的隐藏忠诚度（是否 RAT）。
- 接受任务后被秘密膨胀的 `required_qty`——但你能通过 `task inspect` 看到当前数值，对比 `market browse` 时记下的数值就能识别。

## 你能干的事

整个游戏的动作空间就是这一小撮 CLI 命令（细节见 [07 - Agent 接口](07-agent-interface.md)）：

| 类别 | 你能做的事 |
|------|-----------|
| 观察 | 看公司状态、员工列表、市场任务、自己接了哪些活、客户列表与历史、财务流水 |
| 任务 | 从市场**接受**一个任务、把员工**指派**给任务、**派工**让它开始干、**取消**一个任务 |
| 时间 | 调用 `sim resume` 让时间往前走到下一个事件 |
| 记忆 | 往 scratchpad 里**写 / 追加 / 清空**自己的长期备忘 |

注意：**真正让时间走的只有 `sim resume`**。其他命令再多都不会让时间前进。你可以连续看 10 次状态再决定动作。

## 一回合是什么样的

一回合通常长这样：

```
1. 你（Agent）收到一条系统提示，里面有"上次时间推进后发生了什么"
2. 你想了想，调用若干个观察命令（市场怎么样？我现金还剩多少？）
3. 你做出决策：接哪个任务、派谁去、要不要取消哪个
4. 你调用 sim resume 让时间往前走到下一个有意义的事件
5. 系统返回"这次时间推进期间触发了哪些事件"（任务完成？工资发了？里程碑？破产？到期？）
6. 进入下一回合
```

"下一个事件"可能是几小时后，也可能是十几天后。**时间不是按小时跳的，是按事件跳的**——这就是"离散事件仿真"。引擎在推进过程中遇到月初工资日也会停下来交还控制权（见 [06 - 时间、事件与工资](06-time-and-events.md)）。

> 如果你连续若干回合都没有调用 `sim resume`（默认阈值 `auto_advance_after_turns = 5`），循环会强制帮你 resume 一次。否则你会陷在"分析瘫痪"里把回合数耗光。`max_turns` 默认未启用（在 `default.toml` 里被注释掉），等同于不设硬性回合上限。

## 任务的一生

```
market（在市场上可见）
   │  task accept
   ▼
planned（你接了，但还没开始干）
   │  task assign（给它派员工）
   │  task dispatch（开工！）
   ▼
active（员工正在干活，进度在涨）
   │
   ├──→ completed_success（在截止日期内完成）
   ├──→ completed_fail（超过截止日期才完成）
   └──→ cancelled（你中途放弃了）
```

几个关键点：

- **接受任务的瞬间，截止日期就开始计时了**。所以"先接了再说"很危险。
- **accept 和 dispatch 是分开的**：接完之后你还有一个"派员工"的阶段，不会立刻消耗员工产能。但截止日期已经在走了。
- **任务有多个领域要求**：一个任务可能同时要 60 单位的 research + 30 单位的 training，**所有领域都做满才算完**。
- **任务有声望门槛**：低声望接不到高声望任务。
- **部分任务有信任门槛**：约 30% 的高价任务（`trust_gating_fraction = 0.30`）要求你和发包客户已经攒了一定信任。

## 钱怎么进出

**进账**（让你赚钱的事）：

- 任务**按时**完成：拿到 reward，金额还会被你的声望放大。

**出账**（让你掉钱的事）：

- 每月第一个工作日：**自动扣全员工资**。
- 任务**延期完成**：会扣报酬的 35%（`penalty_fail_funds_pct = 0.35`）作为现金罚款，并应用一份声望惩罚。
- 任务**取消**：在 `default` 预设下没有现金罚款，但会按 `penalty_cancel_multiplier = 1.5` 扣声望。

资金都按"分（cents）"用整数存。**资金 < 0** 的检查只在每月扣完工资那一刻进行，一旦触发就是**破产**——游戏立刻结束。

> 时序很关键：引擎在 `advance_time` 里按"同一时间戳，工资先于事件"的优先级处理。如果工资日和某个任务完成事件落在同一时间点，会**先扣工资 → 立即按当时余额做破产检查**；如果工资扣完已经为负，破产事件会被插入并立刻结束推进。之后的任务报酬要等下次推进才能入账，**所以工资日扣完已经破产，靠"同一时刻完成的任务"是救不回来的**——这一点与一些早期设计描述不同，请以代码为准。

## 声望：你能接什么活的"门票"

每个领域（research、inference、data_environment、training）有一个 1.0 ~ 10.0 的声望值，开局所有领域都从 `initial_prestige_level = 1.0` 起步。

- **任务按时完成** → 相关领域声望上涨。
- **任务延期 / 取消** → 相关领域声望下跌（取消的扣分更重）。
- **被动衰减**默认是**关闭**的（`prestige_decay_per_day = 0.0`），但自定义预设可以打开。

声望直接影响：

1. **你能接什么任务**：每个任务有 `required_prestige`，**你在该任务涉及的每个领域都要达到门槛**才能接（不是平均，是逐一）。
2. **市场上能看到多贵的活**：高 `required_prestige` 的任务在世界生成时报酬就被预先放大（`reward_prestige_scale = 0.30`）；声望升上去 → 解锁更高门槛的任务 → 那些任务列价本身就更高。

详见 [03 - 四个领域与声望](03-four-domains-and-prestige.md)。

## 信任：和客户的"长期合作关系"

每个客户和你之间有一个 0 ~ 5 的信任值，**默认从 0 开始**。

- 完成他的任务 → 信任涨。
- 失败 / 取消 → 信任掉。
- 每天有一个小衰减。
- **完成 A 的任务**会让对 B、C、D 的信任轻微掉一点（"你最近老给别人干活"）——所以你**没法和所有客户都维持高信任**，必须挑 2~3 家重点经营。

信任的好处是：

- **工作量削减**：信任高时，他派给你的任务实际所需工作量最多减少 `trust_work_reduction_max = 50%`。
- **解锁门槛任务**：约 30% 的高价任务（`trust_gating_fraction = 0.30`）需要"信任 ≥ 某值"才能接。

> 注意：信任**不直接放大报酬金额**（`trust_gated_reward_boost = 0.0`），它的收益是间接的——让你单位时间能干更多活。

详见 [05 - 客户、信任与 RAT 客户](05-clients-and-trust.md)。

## 游戏怎么结束

主要有两种正常结局：

| 结局 | 触发条件 |
|------|---------|
| `horizon_end` | 仿真时间走到 `horizon_years`（默认 1 年）的末尾，你**活下来了** |
| `bankruptcy` | 某次发完工资后资金变成负数 |

如果在自定义预设里启用了 `loop.max_turns`（`default.toml` 里默认是注释掉的、不启用），还会有第三种结局：把硬性最大回合数耗光被强制结束。

结局之后，会把这次跑的完整 transcript 和最终状态保存成一份 JSON，用于之后对比分析。

## 一份"理性玩家"的心智模型

如果让一个人来玩，他大致会按这个心智模型走：

1. **前 1~2 个月**：拿 `employee list` 给的 `skill_rates` 表做指派；先做几单低门槛任务把声望刷到 2~3，同时观察哪些客户**反复延期**（早期识别 RAT 候选）。
2. **第 3~4 个月**：挑 2~3 个看起来不像 RAT 的客户重点合作，攒信任，让他们派的任务工作量削减率往上走。
3. **中期**：已识别的 RAT 客户彻底拉黑（不再接他们的任务）；用客户专长偏置去接收益最高的活。
4. **后期**：警惕工资上涨——已"练满"（速率 10 或工资到 $100K 上限）的员工不要再无脑派活；盯住月度跑道避免年底前现金流崩盘。

这只是"一种"打法。基准的意义就在于，**让不同模型自己摸索它们的打法**，然后用客观指标比一比。
