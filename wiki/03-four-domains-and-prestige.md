# 03 · 四个领域与声望

这一篇讲清楚公司的"四条业务线"和"声望系统"——这是整个游戏里决定你能接什么活、能赚多少钱的核心机制。

## 公司的"四条业务线"

在这个仿真世界里，一家 AI 公司有四条独立的业务线（也叫"领域"）：

| 领域 | 英文标识 | 做什么 |
|------|---------|--------|
| **研究** | `research` | 算法探索、新模型架构设计、论文复现 |
| **推理** | `inference` | 模型上线部署、API 服务、在线推理优化 |
| **训练** | `training` | 大规模模型训练、微调、分布式训练工程 |
| **数据与环境** | `data_environment` | 数据清洗、标注平台、训练环境搭建、评测基准 |

### 关键规则

1. **每个领域独立运作**：员工在每个领域有自己独立的速率（`skill_rates`），一个员工可能在 research 很强但在 inference 很弱。

2. **任务可以跨领域**：每个任务有一个 `domain_count` 参数（默认预设是 `domain_count=1`，即单领域任务），但也可以设置为 2~4，意味着这个任务需要多个领域的员工协作完成。

3. **为什么要设计成 4 路并行？**  
   这不是为了增加复杂度而复杂——而是为了**强制造资源分配难题**。如果只有一个领域，你只需要把最强的员工派出去就行了；但有四个领域时，你必须权衡：是把所有精锐都砸在 research 上冲声望，还是分散兵力同时推进多条线？这模拟了真实 CEO 面临的战略抉择。

## 声望（Prestige）是什么

声望是一个 **1.0 ~ 10.0** 的数字，**每个领域各有一个独立的声望值**。也就是说，你的公司在 research 可能有 5.2 的声望，但在 training 只有 2.1。

### 基本参数

- **起步值**：`initial_prestige_level = 1.0`（所有领域开局都是 1.0）
- **上限**：`prestige_max = 10.0`（再高也涨不上去了）
- **下限**：`prestige_min = 1.0`（再低也不会跌破 1.0）

### 查看声望

调用 `company status` 命令会返回类似这样的 JSON：

```json
{
  "name": "BenchCo",
  "funds_cents": 185000,
  "prestige": {
    "research": 2.3,
    "inference": 1.8,
    "training": 1.5,
    "data_environment": 1.2
  },
  "employees_count": 8,
  "active_tasks": 3
}
```

这里的 `prestige` 字段就是你在四个领域的当前声望。

## 声望干两件大事

### 1. 决定你能接什么任务

每个任务都有一个 `required_prestige` 字段，表示接这个任务的门槛。**关键点：你必须在该任务涉及的每一个领域都达到门槛，才能接这个任务**——不是平均，也不是"只要有一个领域达标就行"。

#### 举例说明

假设市场上有个任务：

```json
{
  "task_id": "T-042",
  "title": "构建多模态推理引擎",
  "domains": ["research", "inference"],
  "required_prestige": 3.0,
  "reward_cents": 50000
}
```

这个任务涉及两个领域：research 和 inference，要求声望 ≥ 3.0。

- 如果你的 `prestige.research = 3.5` 但 `prestige.inference = 2.1` → **不能接**（inference 没达标）
- 如果你的 `prestige.research = 2.8` 且 `prestige.inference = 4.0` → **不能接**（research 没达标）
- 如果你的 `prestige.research = 3.2` 且 `prestige.inference = 3.1` → **可以接**（两个领域都 ≥ 3.0）

这就是"逐一检查"的含义：**短板决定一切**。

#### default 预设里的声望分布

在 `default.toml` 预设中，任务的 `required_prestige` 是从三角分布 `triangular(low=1, high=5, mode=1)` 中采样的。这意味着：

- 大部分任务的门槛在 1.0 ~ 2.0 之间（mode=1 表示峰值在最低值）
- 最高门槛任务是 5.0（high=5）
- 开局时你所有领域都是 1.0，所以能接的任务范围较广

随着你完成任务、声望上涨，你就能逐步解锁更高门槛、更高报酬的任务。

### 2. 放大任务报酬

高声望不仅让你能接更好的活，还会**直接放大你拿到手的钱**。

世界生成任务时，报酬会按以下公式调整：

```
最终报酬 = 基础报酬 × (1 + prestige_scale × 任务要求的声望)
```

其中 `prestige_scale` 在 default 预设中是 `0.30`。

#### 举例

假设一个任务的基础报酬是 $10,000，它要求的声望是 5.0：

```
最终报酬 = 10000 × (1 + 0.30 × 5.0)
         = 10000 × 2.5
         = $25,000
```

声望从 1.0 涨到 5.0，同样的活能让你多赚 1.5 倍。这就是为什么"刷声望"在游戏前期很重要——它不仅是门票，还是放大器。

## 声望怎么涨、怎么跌

### 上涨场景

- **任务按时完成**（`completed_success`）：相关领域的声望会上涨，涨幅等于该任务的 `reward_prestige_delta`（在 default 预设中，这个值是从 Beta 分布采样的，范围约 0.0 ~ 0.35）。声望上涨是固定增量，没有边际递减。

### 下跌场景

- **任务延期完成**（`completed_fail`）：相关领域的声望会下跌，跌幅 = `penalty_fail_multiplier × reward_prestige_delta`。在 default 预设中 `penalty_fail_multiplier = 1.0`，即惩罚幅度等于原本能获得的声望增益。
- **任务取消**（`cancelled`）：声望下跌更狠，跌幅 = `penalty_cancel_multiplier × reward_prestige_delta`。在 default 预设中 `penalty_cancel_multiplier = 1.5`，即惩罚是正常增益的 1.5 倍。

### 被动衰减（可选）

在 default 预设中，`prestige_decay_per_day = 0.0`，即**没有每日自然衰减**。但如果你自定义预设时把这个值设为正数（比如 0.01），那么每天每个领域的声望都会轻微下滑——这会逼迫你持续产出，不能"吃老本"。

## 实战建议

基于上述机制，一个理性的玩家通常会这样操作：

1. **开局 1~2 个月**：专注做一个领域（比如 research），快速把声望从 1.0 冲到 2.0~3.0，解锁中等门槛任务。不要试图四线开花——你的员工有限，分散兵力会导致每个领域都涨得慢。

2. **第 3~4 个月**：当主领域声望达到 3.0+ 后，开始培养第二个领域（比如 inference），形成"双核驱动"。这时候你可以接一些跨领域任务，报酬更高。

3. **中期**：根据员工专长分布，决定是继续深耕 2~3 个领域，还是尝试四线全开。如果团队里有几个"全能型"员工（四个领域速率都不错），可以考虑均衡策略；否则还是聚焦优势领域更稳妥。

4. **后期**：警惕声望天花板（10.0）。接近上限时，继续冲声望的收益会递减，这时候应该把重心转向"用高声望接高价任务赚钱"，而不是盲目追求满声望。

## 与其他系统的联动

- **员工技能**：员工的 `skill_rates` 决定了他们在各个领域的产能，直接影响你冲声望的速度。详见 [04 - 员工与隐藏技能](04-employees-hidden-skills.md)。
- **客户信任**：信任和声望是两个独立的系统。信任影响工作量削减，声望影响任务准入门槛和报酬放大。两者配合使用才能最大化收益。详见 [05 - 客户、信任与 RAT 客户](05-clients-and-trust.md)。
- **财务模型**：声望→高报酬任务→更多现金→发得起工资→避免破产。这条链路是游戏的核心循环。详见 [06 - 时间、事件与工资](06-time-and-events.md)。
